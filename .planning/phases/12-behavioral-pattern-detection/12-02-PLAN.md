import { pool } from '../db';

`detectRepeatGoingDark()` implementation:
1. Query `vessel_anomalies` for IMOs with 3+ going_dark events (both active AND resolved) in the last 30 days:
   ```sql
   SELECT imo, COUNT(*) as dark_count,
          json_agg(json_build_object(
            'detectedAt', detected_at,
            'resolvedAt', resolved_at
          ) ORDER BY detected_at DESC) as recent_events
   FROM vessel_anomalies
   WHERE anomaly_type = 'going_dark'
     AND detected_at > NOW() - INTERVAL '30 days'
   GROUP BY imo
   HAVING COUNT(*) >= 3
   ```
2. For each result, call `upsertAnomaly()` with:
   - `anomalyType: 'repeat_going_dark'`
   - `confidence: 'confirmed'`
   - `details: { goingDarkCount: dark_count, windowDays: 30, recentEvents: recent_events }`
3. Auto-resolve: after the upsert loop, resolve any existing `repeat_going_dark` anomalies for IMOs NOT in the result set:
   ```sql
   UPDATE vessel_anomalies SET resolved_at = NOW()
   WHERE anomaly_type = 'repeat_going_dark' AND resolved_at IS NULL
     AND imo NOT IN (SELECT imo FROM vessel_anomalies
                     WHERE anomaly_type = 'going_dark'
                       AND detected_at > NOW() - INTERVAL '30 days'
                     GROUP BY imo HAVING COUNT(*) >= 3)
   ```
4. Return count of anomalies detected.

**File 2: `src/lib/detection/sts-transfer.ts`**

Create STS transfer detector:

```typescript
import { pool } from '../db';
import { upsertAnomaly } from '../db/anomalies';
import type { StsTransferDetails } from '../../types/anomaly';
```

`detectStsTransfers()` implementation:
1. SQL pair-finding query using haversine formula — find vessel pairs within 0.926km (0.5nm) where both have positions in the last 30 minutes:
   ```sql
   SELECT DISTINCT ON (LEAST(a.imo, b.imo), GREATEST(a.imo, b.imo))
     a.imo as imo_a, a.name as name_a, a_pos.latitude as lat_a, a_pos.longitude as lon_a,
     b.imo as imo_b, b.name as name_b, b_pos.latitude as lat_b, b_pos.longitude as lon_b,
     2 * 6371 * asin(sqrt(
       sin(radians((b_pos.latitude - a_pos.latitude) / 2))^2 +
       cos(radians(a_pos.latitude)) * cos(radians(b_pos.latitude)) *
       sin(radians((b_pos.longitude - a_pos.longitude) / 2))^2
     )) as distance_km
   FROM vessels a
   JOIN LATERAL (
     SELECT latitude, longitude FROM vessel_positions
     WHERE mmsi = a.mmsi AND time > NOW() - INTERVAL '30 minutes'
     ORDER BY time DESC LIMIT 1
   ) a_pos ON true
   JOIN vessels b ON b.imo > a.imo
   JOIN LATERAL (
     SELECT latitude, longitude FROM vessel_positions
     WHERE mmsi = b.mmsi AND time > NOW() - INTERVAL '30 minutes'
     ORDER BY time DESC LIMIT 1
   ) b_pos ON true
   WHERE 2 * 6371 * asin(sqrt(
       sin(radians((b_pos.latitude - a_pos.latitude) / 2))^2 +
       cos(radians(a_pos.latitude)) * cos(radians(b_pos.latitude)) *
       sin(radians((b_pos.longitude - a_pos.longitude) / 2))^2
     )) < 0.926
   ```
   Note: `b.imo > a.imo` ensures each pair is only found once (lexicographic deduplication).

2. For each pair, create TWO anomalies (one for each vessel):
   - Vessel A: `{ otherImo: imo_b, otherName: name_b, distanceKm, lat: lat_a, lon: lon_a }`
   - Vessel B: `{ otherImo: imo_a, otherName: name_a, distanceKm, lat: lat_b, lon: lon_b }`
   - Both with `anomalyType: 'sts_transfer'`, `confidence: 'suspected'`

3. Return total count (2 per pair).
