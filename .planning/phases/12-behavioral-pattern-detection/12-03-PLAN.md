```sql
-- Vessel proximity events — tracks when pairs are first observed within 0.5nm
-- Used by STS transfer detector to enforce 30-minute sustained co-location (PATT-03)
CREATE TABLE IF NOT EXISTS vessel_proximity_events (
  imo_a TEXT NOT NULL,
  imo_b TEXT NOT NULL,
  first_seen_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  last_seen_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (imo_a, imo_b)
);
```

Convention: `imo_a < imo_b` lexicographically (enforced by application code via `b.imo > a.imo` join). No separate index needed — PK covers lookups.

1. Keep the existing constants and imports. Add a new constant:
   ```typescript
   const SUSTAINED_PROXIMITY_MINUTES = 30;
   ```

2. After the existing pair-finding query returns `result.rows`, replace the current for-loop with this three-step process:

   **Step A — Upsert proximity events for all currently-close pairs:**
   For each row in result.rows, upsert into `vessel_proximity_events`:
   ```typescript
   for (const row of result.rows) {
     await pool.query(`
       INSERT INTO vessel_proximity_events (imo_a, imo_b, first_seen_at, last_seen_at)
       VALUES ($1, $2, NOW(), NOW())
       ON CONFLICT (imo_a, imo_b) DO UPDATE SET last_seen_at = NOW()
     `, [row.imo_a, row.imo_b]);
   }
   ```
   Note: imo_a < imo_b is already guaranteed by the `b.imo > a.imo` join condition.

   **Step B — Clean up stale proximity events (pairs no longer within 0.5nm):**
   Build a set of currently-close pair keys from result.rows. Delete any rows in `vessel_proximity_events` whose `last_seen_at` is older than the position freshness window (they were NOT updated in Step A, meaning they are no longer close):
   ```typescript
   await pool.query(`
     DELETE FROM vessel_proximity_events
     WHERE last_seen_at < NOW() - INTERVAL '${POSITION_FRESHNESS_MINUTES + 5} minutes'
   `);
   ```
   Use `POSITION_FRESHNESS_MINUTES + 5` (35 minutes) as the staleness threshold to account for slight cron timing drift.

   **Step C — Fire anomalies only for pairs with 30+ minutes of sustained proximity:**
   Query `vessel_proximity_events` for pairs that have been tracked long enough:
   ```typescript
   const sustained = await pool.query<{ imo_a: string; imo_b: string }>(`
     SELECT imo_a, imo_b FROM vessel_proximity_events
     WHERE last_seen_at - first_seen_at >= INTERVAL '${SUSTAINED_PROXIMITY_MINUTES} minutes'
   `);
   ```
   Build a Set of sustained pair keys (`${imo_a}:${imo_b}`). Then iterate result.rows and only call `upsertAnomaly` for pairs that appear in the sustained set:
   ```typescript
   const sustainedPairs = new Set(sustained.rows.map(r => `${r.imo_a}:${r.imo_b}`));
   let count = 0;

   for (const row of result.rows) {
     const pairKey = `${row.imo_a}:${row.imo_b}`;
     if (!sustainedPairs.has(pairKey)) continue;

     // ... existing upsertAnomaly calls for vessel A and vessel B (keep as-is)
     count += 2;
   }
   ```
