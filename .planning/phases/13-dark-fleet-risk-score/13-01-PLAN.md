    **2. Create src/lib/db/risk-scores.ts** — DB operations

    Two exported functions:

    `upsertRiskScore(imo: string, score: number, factors: RiskFactors): Promise<void>` — INSERT ON CONFLICT (imo) DO UPDATE SET score, factors, computed_at = NOW()

    `getRiskScore(imo: string): Promise<{ score: number; factors: RiskFactors; computedAt: string | null }>` — SELECT from vessel_risk_scores WHERE imo = $1. If no row found, return `{ score: 0, factors: { goingDark: 0, flagRisk: 0, sanctions: 0, loitering: 0, sts: 0 }, computedAt: null }`.

    Define and export:
    ```typescript
    export interface RiskFactors {
      goingDark: number;   // max 40
      flagRisk: number;    // max 15
      sanctions: number;   // max 25
      loitering: number;   // max 10
      sts: number;         // max 10
    }
    ```

    Import `pool` from `./index`.

    **3. Create src/lib/detection/risk-score.ts** — Computation logic

    Export `async function computeRiskScores(): Promise<number>` (returns count of vessels scored).

    Implementation:
    1. Query all distinct IMOs from vessel_anomalies (any anomaly_type, any status).
    2. For each IMO, compute factors:

       **goingDark** (max 40): Count going_dark anomalies (active + resolved, all time). Score = Math.min(count * 8, 40).

       **flagRisk** (max 15): Join vessels table for flag. High-risk flags: `['IR', 'RU', 'VE', 'KP', 'PA', 'CM', 'KM']` (Iran, Russia, Venezuela, North Korea, Panama, Cameroon, Comoros). Score = 15 if flag in list, else 0.

       **sanctions** (max 25): Check if IMO has any active anomaly where anomaly_type includes sanctions-related data OR exists in vessel_sanctions table. Simplest: `SELECT 1 FROM vessel_sanctions WHERE imo = $1`. Score = 25 if match, else 0.

       **loitering** (max 10): Count loitering anomalies in last 90 days (active + resolved). Score = 10 if count > 0, else 0. Binary.

       **sts** (max 10): Count sts_transfer anomalies (all time, active + resolved). Score = 10 if count > 0, else 0. Binary.

    3. total score = goingDark + flagRisk + sanctions + loitering + sts (capped at 100 by design since 40+15+25+10+10=100).
    4. Call upsertRiskScore(imo, score, factors) for each vessel.
    5. Return count of vessels processed.

    For efficiency, use a SINGLE SQL query that aggregates all factors at once rather than N+1 queries per vessel:

    ```sql
    SELECT
      va.imo,
      v.flag,
      COUNT(*) FILTER (WHERE va.anomaly_type = 'going_dark') as dark_count,
      COUNT(*) FILTER (WHERE va.anomaly_type = 'loitering' AND va.detected_at > NOW() - INTERVAL '90 days') as loiter_count,
      COUNT(*) FILTER (WHERE va.anomaly_type = 'sts_transfer') as sts_count,
      CASE WHEN vs.imo IS NOT NULL THEN 1 ELSE 0 END as is_sanctioned
    FROM vessel_anomalies va
    LEFT JOIN vessels v ON v.imo = va.imo
    LEFT JOIN vessel_sanctions vs ON vs.imo = va.imo
    GROUP BY va.imo, v.flag, vs.imo
    ```

    Then compute scores in JS from the query result. Import `pool` from `../db` and `upsertRiskScore` from `../db/risk-scores`.

    ```typescript
    /**
     * GET /api/vessels/[imo]/risk - Returns dark fleet risk score for a vessel.
     * Requirements: RISK-01
     */
    import { NextResponse } from 'next/server';
    import { getRiskScore } from '@/lib/db/risk-scores';

    export async function GET(
      request: Request,
      { params }: { params: Promise<{ imo: string }> }
    ) {
      const { imo } = await params;

      try {
        const risk = await getRiskScore(imo);
        return NextResponse.json(risk);
      } catch (error) {
        console.error(`[API] Error fetching risk score for ${imo}:`, error);
        return NextResponse.json(
          { error: 'Failed to fetch risk score' },
          { status: 500 }
        );
      }
    }
    ```
